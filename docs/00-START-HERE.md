# Start Here

## Welcome to the Agent Methodology Pack

This documentation follows the BMAD structure for organized project management.

## Documentation Structure

### 1-BASELINE
Foundational documents:
- **product/** - PRD, personas, requirements
- **architecture/** - System design, ADRs, schemas
- **research/** - Research findings, analysis

### 2-MANAGEMENT
Project management:
- **epics/** - Epic specifications and stories
  - current/ - Active epics
  - completed/ - Finished epics

### 3-ARCHITECTURE
Design artifacts:
- **ux/** - User experience documentation
  - flows/ - User flows
  - wireframes/ - UI wireframes
  - specs/ - UI specifications

### 4-DEVELOPMENT
Development documentation:
- Implementation guides
- API documentation
- Development notes

### 5-ARCHIVE
Historical documents:
- Completed sprints
- Old versions
- Reference material

## Getting Started

1. Read the PRD in `1-BASELINE/product/`
2. Review architecture in `1-BASELINE/architecture/`
3. Check current work in `2-MANAGEMENT/epics/current/`
4. Track progress in `PROJECT-STATE.md`
